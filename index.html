# Spec: Fix PDF Export Quality (Maisoku Generator) ## Context Single-file HTML app (`index.html`, ~2050 lines, no build step, no npm). It generates A4-landscape Japanese real-estate sheets (マイソク). The live preview renders perfectly, but PDF export via `html2canvas` produces blurry, stretched, low-resolution output — photos and text degrade badly. **Root cause:** `html2canvas` is a screenshot library. It rasterizes the entire sheet into one flat JPEG, then `jsPDF` scales that JPEG to A4 — flattening sharp text into blurry pixels and re-compressing already-rendered photos. It also re-lays-out the DOM in a hidden iframe, so proportions drift from the preview. **The fix:** Replace the html2canvas export path with the **browser's native print-to-PDF engine**, which renders the HTML at true vector quality — sharp text, native-resolution photos, exact A4 sizing. The print path partially exists already; this spec makes it the primary, correct path. --- ## Key facts about the current code - The maisoku sheet element is `#ms`, fixed at **`width:1047px; height:740px`** (CSS line ~78). This aspect ratio (1.415) already matches A4 landscape (297:210 = 1.414). **Do not change these dimensions.** - There is already an `@media print` block (CSS line ~156) and an `@page{size:A4 landscape;margin:0}` rule. - `doExportPDF()` (line ~1453) currently uses html2canvas + jsPDF. **This is what we are replacing.** - `doPrint()` (line ~1444) uses a `transform: scale()` hack before `window.print()` — **this hack is buggy** (the print engine doesn't reflow the page box around a CSS-transformed element, causing clipping). It must be removed. - The export button is `#pdfBtn` (HTML line ~180), label driven by `T[lang].btnPdf` ("One-Click PDF Export" / "ワンクリックPDF出力"). - There is also a System Print button `#printBtn` calling `doPrint()`. - html2canvas and jspdf are loaded via CDN (lines ~7). --- ## Required changes ### 1. Rewrite the `@media print` CSS block Replace the existing block (CSS ~156–164) so that when printing: - Only `#ms` is visible. Hide `#topbar`, `#sidebar`, `#fp`, `#pp .phint`, `.topbtns`, and any toast/warning divs. - `#ms` renders at its natural `1047×740px` but is scaled to fill the A4 page via the page box, NOT via CSS `transform`. Use this approach: set `#ms` to `width: 297mm; height: 210mm;` **inside the print media query only**, overriding the px dimensions, so the browser maps it 1:1 to the A4 page. All child elements use %/flex so they reflow proportionally. - `@page { size: A4 landscape; margin: 0; }` - Force color printing: `* { -webkit-print-color-adjust: exact !important; print-color-adjust: exact !important; }` - Remove `#ms` borders, shadows, and any `position:absolute` offset that could clip it. Set `#ms { position: relative; top: 0; left: 0; margin: 0; }` in print context. - Ensure `#app`, `#pp`, `body`, `html` have no max-width, padding, or overflow clipping in print context. **Critical:** test that overriding `#ms` to `297mm × 210mm` doesn't distort children. Because the px and mm aspect ratios match (both ≈1.414), children sized in % will scale cleanly. If any child uses fixed px that breaks, convert it to % or leave `#ms` in px and instead rely on `@page` + the browser "Fit to page" — but the mm override is preferred because it removes dependence on user print settings. ### 2. Remove the `transform: scale()` hack from `doPrint()` Rewrite `doPrint()` to simply: ``` function doPrint(){ window.print(); } ``` No transform, no scale calc, no setTimeout cleanup. The print CSS now handles sizing entirely. ### 3. Repoint "One-Click PDF Export" to the print engine Rewrite `doExportPDF()` so it no longer uses html2canvas/jsPDF. It should: - Optionally show a brief one-time hint (a small dismissable tooltip) telling the user: "In the print dialog, choose 'Save as PDF', set Margins to 'None', and enable 'Background graphics'." (EN) / appropriate JA equivalent. - Then call `window.print()`. - Keep the function name `doExportPDF` so the existing button `onclick` still works. ### 4. Keep html2canvas as a silent fallback ONLY (optional) - Do NOT delete the CDN includes (harmless if unused). - If you want a true one-click download path retained, keep a separate hidden/secondary function, but the PRIMARY `doExportPDF` must use `window.print()`. Do not call html2canvas in the default path. ### 5. Photo rendering consistency in print - Verify every property/photo `` and image container uses `object-fit: cover` (for photos that fill a box) or `object-fit: contain` (for floorplans/logos that must not crop) consistently, so nothing distorts when the sheet maps to A4. - The map tile container and uploaded map image must also use the same object-fit rules. - Confirm `print-color-adjust: exact` is applied so the red header cells and colored text render in the PDF. --- ## Acceptance criteria 1. Clicking "One-Click PDF Export" opens the browser print dialog (not a direct blurry download). 2. Choosing "Save as PDF" with margins None produces a PDF where: - Text is sharp (vector, not pixelated). - Property photos are at native resolution (no blur, no stretch). - The sheet fills exactly one A4 landscape page with no white margins and no clipping. - Red header cells and colored text appear (color graphics on). - Layout matches the on-screen preview exactly (same proportions, no squashing). 3. "System Print" button also works via the same clean `window.print()` path. 4. No `transform: scale()` is used anywhere in the print flow. 5. App remains a single standalone HTML file — no new dependencies, no build step. --- ## Constraints - Single-file `index.html`. No npm, no bundler, no server. - CDN-only dependencies (existing ones may stay). - Must work in Chrome (primary) and Safari (secondary). - Do not change the `#ms` base px dimensions (1047×740) outside the print media query. - Do not alter the data model, AI scan logic, amenities logic, or translation system — this task is print/export only.
